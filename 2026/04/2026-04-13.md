  - `tests/models/test_adapters.py`
  - `vllm/model_executor/models/adapters.py`

### Summary

**What changed and why**  
The fix addresses silent weight corruption when using buffer-reusing iterators (e.g., `runai_streamer` with `MEMORY_LIMIT=0`). The original `(*seen_weights, *weights)` eagerly consumed the entire iterator, causing shared I/O buffers to be overwritten before tensors were used. The solution replaces this with `itertools.chain` for lazy iteration and adds `.clone()` to probed weights to preserve data during packed-weight remapping.

**Technical impact**  
These changes ensure that weight loading remains safe with lazy, buffer-reusing iterators, preventing data corruption in pooling/embedding models. The lazy chaining defers tensor consumption, while cloning probed weights ensures that remapped weights (e.g., q/k/v_proj → qkv_proj) are not overwritten during iteration.

**Potential risks**  
Cloning every probed weight introduces a small memory and performance overhead, though it is necessary for correctness. If the iterator yields very large tensors, cloning could temporarily double memory usage. Additionally, the fix assumes that all iterators benefit from lazy chaining; however, this is a safe generalization.

**Key insights**  
Always use lazy iteration (`itertools.chain`) when combining iterators in weight-loading code to avoid eager consumption. Clone tensors from mutable buffers immediately to prevent data corruption during remapping or deferred processing. The added tests provide robust coverage for both buffer reuse and packed-weight scenarios.
