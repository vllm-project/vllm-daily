  - `vllm/v1/kv_offload/mediums.py`
  - `vllm/v1/kv_offload/worker/cpu_gpu.py`

### Summary

**What changed and why**  
This PR extends the CPU-GPU offloading handler to support transfers with multiple KV cache groups, which is needed for hybrid model architectures (HMA). The changes include: (1) removing the single-group assertion in the handler, (2) updating the transfer logic to iterate over groups and compute per-group sub-block pointers, (3) making block_indices a required field in GPULoadStoreSpec, (4) updating the scheduler to pass block_indices, and (5) adding comprehensive tests for multi-group transfers including empty groups and unaligned transfers.

**Technical impact**  
The offloading handler now correctly handles multiple KV cache groups, each with potentially different sizes and alignment offsets. The transfer_async method computes copy operations per group, using group_sizes and block_indices from GPULoadStoreSpec to determine the correct sub-block offsets. The total transfer size is computed as the sum over groups of group_size * page_size_bytes per tensor. The scheduler is updated to pass block_indices=(0,) for single-group transfers. Tests cover aligned, empty, and unaligned groups.

**Potential risks**  
The change makes block_indices mandatory in GPULoadStoreSpec, which could break external code that previously passed None. The new multi-group logic assumes group_sizes and block_indices align with kv_cache_groups_data_refs; mismatches could cause out-of-bounds access. The compute_sub_block_ptrs function is called per group and per tensor, which may increase overhead for many groups/tensors. The test coverage is extensive but may miss edge cases like very large numbers of groups or zero-length groups.

**Key insights**  
The PR introduces a systematic way to handle per-group alignment by using block_indices to compute logical offsets. The use of cdiv for block count calculation ensures correct handling of partial blocks. The removal of the single-group assertion and the refactoring of block size computation into per-group loops are the core changes. The test design with three groups (aligned, empty, unaligned) is a good pattern for verifying multi-group support.

The bug occurred because quantized models can have integer dtypes for offloaded parameters, but the code assumed all dtypes were floating-point.
