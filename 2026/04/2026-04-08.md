**What changed and why**  
The changes optimize redundant device synchronization for pooling models by introducing a helper method `_get_or_create_async_output_copy_stream()` and restructuring the `_pool()` method. **Key insights**

The post-processing is not supported with Heterogeneous Multi-head Attention (HMA), which could lead to runtime errors if both features are enabled. Additionally, the packing logic assumes specific tensor shapes and may fail if the remote KV cache layout differs unexpectedly. The lazy import of `cpu_attn` modules could cause initialization issues in edge cases.

**Key insights**  
Developers should note that this feature is experimental and requires careful validation when used with other advanced features like HMA. The packing step adds computational overhead, which may impact performance in high-throughput scenarios. Ensure that the remote attention backend’s KV layout matches the expected plain format to avoid silent accuracy degradation.
