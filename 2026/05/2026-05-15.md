**Technical impact**  
Ensures correctness of layerwise reload for models like Mamba/Nemotron that use non-persistent alias buffers. Prevents NaN generations after repeated checkpoint reloads. The change is minimal (2 lines in layerwise.py, 38 lines in meta.py) and only affects the reload path.

**Potential risks**  
Low risk: the alias detection is conservative (only skips buffers that share storage with a parameter), and the copy-back skip only applies when the buffer is missing. Existing non-alias buffers are unaffected. The new code handles edge cases like uninitialized parameters gracefully.

**Key insights**  
The bug stemmed from two insufficient boundaries: metadata capture only checked direct _parameters instead of recursive parameters, and copy-back assumed all captured buffers exist in the materialized layer. The fix uses storage pointer comparison to detect aliases and gracefully handles missing buffers during copy-back.

  - `vllm/v1/core/kv_cache_coordinator.py`
  - `vllm/v1/core/single_type_kv_cache_manager.py`

### Summary

**What changed and why**  
Introduced a mechanism to skip caching SWA blocks that can never serve a prefix-cache hit in hybrid models (e.g., DeepSeek-V4). The change adds an `alignment_tokens` parameter threaded through `cache_blocks` -> `SlidingWindowManager._cache_block_mask` -> `BlockPool.cache_full_blocks`. A new `_cache_block_mask` method in `SlidingWindowManager` computes which blocks within each alignment-aligned segment are reachable by SWA's right-to-left scan; unreachable blocks are excluded from the prefix-cache hash map. The `HybridKVCacheCoordinator.cache_blocks` method clamps `num_computed_tokens` to the LCM block size and passes it as `alignment_tokens`. Non-hybrid coordinators pass `None` and hit a fast path with no behavior change.

**Technical impact**  
Reduces memory pollution in the prefix-cache hash map and frees LRU list entries for blocks that can never be cache hits. For hybrid models with SWA layers, this prevents caching of blocks that are unreachable due to sliding window constraints, improving cache efficiency and potentially reducing memory pressure. The change is backward-compatible for non-hybrid models via the `None` fast path.

**Potential risks**  
The mask computation in `_cache_block_mask` assumes `alignment_tokens > self.block_size` and uses `cdiv`; edge cases (e.g., `sliding_window <= block_size`) should be verified. The new `block_mask` parameter in `cache_full_blocks` adds complexity; incorrect mask length or logic could cause assertion failures or incorrect caching. Tests cover two scenarios but may not cover all combinations of block sizes and sliding windows. The change only affects hybrid coordinators; non-hybrid paths are unchanged.

**Key insights**  
The optimization leverages the fact that SWA's `find_longest_cache_hit` only returns blocks from the tail of each alignment-aligned segment, so earlier blocks can never be hits. The mask is computed per segment using `per_segment` and `tail` values derived from `alignment_tokens` and `sliding_window`. The implementation is cleanly layered: the coordinator clamps to LCM, the manager computes the mask, and the pool applies it. Tests validate both the tail-window-only and LCM-clamping behaviors.
