**Technical impact**  
Fixes a correctness bug where CUDA graph replay used a stale work-partitioning plan from warmup, causing gibberish output. The root cause was a subtle interaction between CUDA graph warmup and lazy initialization in the FlashMLA kernel. The warmup pass populated the metadata with dummy values, causing the capture pass to skip the planning kernel. By creating fresh state for capture, the planning kernel is always recorded. This fix also protects against similar issues in other backends with lazy initialization.

**Technical impact**

**Key insights**  
The fusion leverages the observation that in GatedDeltaNetAttention, the per-head RMSNormGated output is immediately reshaped and group-quantized, allowing a single fused kernel. The pattern matcher uses dynamic shape inference and static context to avoid hardcoding model parameters.
