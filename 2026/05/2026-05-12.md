- **[Changed Files](https://github.com/vllm-project/vllm/pull/42460/files) (1):**
  - `vllm/model_executor/layers/attention/mla_attention.py`

### Summary

**What changed and why**  
Optimized memory allocation in MLA prefill context computation by reusing two pre-allocated tensors (output and merge_output) instead of allocating a new temporary tensor for each merge iteration. This reduces the number of allocations from O(n) to O(1) per prefill context, significantly lowering memory overhead.

**Technical impact**  
Reduces total allocated memory by up to 94% in the example case (from ~511 MB to ~34 MB) and decreases allocation count from 30 to 2. This improves performance and reduces memory pressure, especially for large chunked prefill contexts.

**Potential risks**  
Low risk. The change is straightforward and only affects the allocation pattern; the merge logic remains identical. The swap of output and merge_output pointers ensures correct behavior. Existing unit tests cover this code path.

**Key insights**  
The optimization leverages the fact that merge_attn_states writes its output to a separate buffer, allowing reuse of two buffers in a ping-pong fashion. This is a classic double-buffering technique applied to reduce dynamic memory allocation overhead.
