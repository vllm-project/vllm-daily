### Summary

**What changed and why**  
The change adds a platform check to skip LD_PRELOAD library validation (for `libtcmalloc` and `libiomp`) on non-Linux systems. This fixes a bug where the check incorrectly ran on macOS, causing failures as shown in the linked CI log.

**Technical impact**  
The validation logic is now restricted to Linux environments, preventing unnecessary or incompatible checks on macOS. This maintains performance optimizations via preloaded libraries on Linux while avoiding runtime errors on other platforms.

**Potential risks**  
If the code is deployed on other non-Linux Unix-like systems (e.g., BSD), similar issues may arise since the check only excludes macOS implicitly. Additionally, any future platform-specific library dependencies would require similar guards.

**Key insights**  
Always condition platform-specific operations (like LD_PRELOAD) on explicit OS checks. Consider extending the guard to `sys.platform != "linux"` if other non-Linux platforms should also skip these checks. Ensure CI tests cover all supported platforms to catch similar issues early.

**Technical impact**
