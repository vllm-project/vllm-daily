## 33. [[EPLB] Remove main waits in case of slow EPLB](https://github.com/vllm-project/vllm/pull/36271)


### Base Information

- **PR Number:** #36271
- **Author:** [ilmarkov](https://github.com/ilmarkov)
- **Merged By:** [tlrmchlsmth](https://github.com/tlrmchlsmth)
- **Merged time:** 2026-03-24 04:50:44
- **Type:** `SUBSTANTIAL`
- **[Changed Files](https://github.com/vllm-project/vllm/pull/36271/files) (2):**
  - `vllm/distributed/eplb/async_worker.py`
  - `vllm/distributed/eplb/eplb_state.py`

### Summary

**What changed and why**  
The PR removes synchronization between the main thread and EPLB async thread using CUDA events. Previously, the main thread waited for weight transfer scheduling on the EPLB stream, which could block main stream execution if transfers were slow. Now, the async thread synchronizes after transfers complete before notifying the main thread, reducing performance impact on the main stream.

**Technical impact**  
This change decouples the main execution stream from EPLB transfer latency by moving the synchronization point from event-based waiting to thread-level notification. The main thread no longer waits on `buffer_ready_event`, eliminating potential stalls when EPLB transfers are slow. The architecture shifts from stream-level synchronization to thread-level coordination.

**Potential risks**  
Removing the CUDA event wait could introduce race conditions if the main thread accesses buffers before transfers complete, though the `ep_buffer_ready` flag and locks should mitigate this. There's a risk of increased memory pressure if the async thread holds buffers longer while synchronizing. The change assumes the async thread's `cuda_stream.synchronize()` is sufficient for correctness.

**Key insights**  
This optimization targets latency-sensitive workloads by isolating main stream execution from EPLB transfer variability. Developers should verify that the `ep_buffer_ready` flag and buffer locking mechanisms provide adequate synchronization. Consider monitoring for any edge cases where buffer access timing might be affected by the removed event wait.
